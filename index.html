MVP · Архитектура решения

Архитектура MVP платформы онлайн-дистрибуции автокомпонентов

Первый этап строится вокруг одного backend-ядра: каталог, складской учёт, заказы, резерв, доставка, оплата, B2C/B2B кабинеты и админская часть. Инфраструктура остаётся без лишнего оверхеда: Docker Compose, внешний S3, PostgreSQL, frontend на SvelteKit.

Списание

Безопасность

  • Отдельная авторизация в админку
  • Роли и права доступа
  • Хеширование паролей
  • Защита admin routes через plugin / middleware слой

Инфраструктура

  • Polyrepo: infra / backend / frontend
  • Docker Compose для dev и production старта
  • Caddy как proxy / SSL
  • Без Kubernetes на первом этапе

Наблюдаемость

  • Pino + Docker/file logs на старте
  • Smoke checks после деплоя
  • Grafana / Prometheus позже, по мере роста
  • Без тяжёлого monitoring-стека на MVP

Комментарий по архитектурному подходу

На первом MVP этапе архитектура специально остаётся без лишней инфраструктурной сложности: единое backend-ядро, PostgreSQL как основная БД, внешний S3 для файлов, Docker Compose, отдельный frontend и базовый кэш/in-memory слой. Такой подход позволяет быстрее выйти в рабочий продукт, а усложнение - очереди, расширенный monitoring, более тяжёлый production-контур - добавлять уже после запуска.
